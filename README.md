# Arduino API Wrappers for STM32CubeIDE

This repository provides Arduino-like wrapper APIs for:

- `gpio`: `pinMode`, `digitalWrite`, `digitalRead`
- `i2c`: `Wire` (`TwoWire`)
- `spi`: `SPI` (`SPIClass`)

## 1. Add Files to CubeIDE Project

Add these files to your CubeIDE project:

- `arduino_compat_common.h`
- `gpio/arduino_gpio.h`, `gpio/arduino_gpio.c`
- `i2c/arduino_wire.h`, `i2c/arduino_wire.cpp`
- `spi/arduino_spi.h`, `spi/arduino_spi.cpp`

Ensure C++ compilation is enabled for `.cpp` files.

## 2. Initialize in `main.c` / `main.cpp`

```c
#include "gpio/arduino_gpio.h"
```

```cpp
#include "i2c/arduino_wire.h"
#include "spi/arduino_spi.h"
```

After CubeMX init (`MX_GPIO_Init`, `MX_I2C1_Init`, `MX_SPI1_Init`):

```c
Arduino_GPIO_ClearPinMap();
Arduino_GPIO_RegisterPin(13, GPIOC, GPIO_PIN_13); // example: LED pin
pinMode(13, OUTPUT);
digitalWrite(13, HIGH);
```

```cpp
extern I2C_HandleTypeDef hi2c1;
extern SPI_HandleTypeDef hspi1;

Wire.attachHandle(&hi2c1);
Wire.begin();

SPI.attachHandle(&hspi1);
SPI.begin();
```

## Notes

- `Wire.setClock()` is a no-op by default because timing is typically generated by CubeMX.
- `SPISettings` updates polarity/phase/bit order and re-initializes HAL SPI.
- SPI clock prescaler remains CubeMX default in this minimal wrapper.
